# Farm Recorder — Mobile Application

An offline-first Flutter mobile application for farm workers to record field activities, scan input barcodes, capture image/video evidence, and view daily task assignments.

## 📱 Core Architecture & Features

### 1. Offline-First Caching (Isar Database)
- Uses **Isar** as a high-performance local database.
- Caches agricultural inputs, approved chemicals, and active inventory batches locally.
- **Offline Activity Queue**: Stashes logged activities and media locally as `OfflineActivityLog` and `OfflineMedia` when network coverage is lost.

### 2. Barcode Scanning & Auto-Prepopulation
- Uses the device camera to scan GS1 barcodes.
- **GS1 Barcode Parser**: Extracts Application Identifiers (AI) including GTIN (`01`), Lot Number (`10`), and Expiration Date (`17`).
- Queries local cached inventory. If a match is found:
  - Auto-prepopulates chemical expiration dates.
  - Resolves chemical brand details and displays stock levels ("In stock: X Liters").

### 3. Media Upload (MinIO Presigned URLs)
- Uploads images and videos directly to MinIO/S3 using secure presigned URLs generated by the backend.
- Links uploaded media assets to local activity logs securely.

### 4. Background Sync Service (`SyncService`)
- Monitored network connectivity changes.
- Automatically drains staged offline activities and uploads media files once network connection is recovered.
- Refreshes local product registries and active stock lots.

## 🛠️ Development & Environment Setup

### Prerequisites
1. Installed **Flutter SDK (v3.x+)**.
2. Run the Quarkus backend API server locally.

### Generated Source Files
Run the build runner to generate Isar database adapters:
```bash
dart run build_runner build --delete-conflicting-outputs
```

### Running the App
Run the app on your connected device or emulator:
```bash
flutter run
```

### Run Tests
Execute the unit and widget tests:
```bash
flutter test
```
